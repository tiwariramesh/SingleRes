## ğŸ—ï¸ Technology Stack

### Frontend
- **Framework**: **React 18+** with **Vite** (fast build tool, modern dev experience)
- **UI Library**: **shadcn/ui** (built on Radix UI + Tailwind CSS)
  - Pre-built, accessible components
  - Highly customizable
  - Free and open-source
  - Excellent for consistency and responsiveness
- **Styling**: **Tailwind CSS** (utility-first, matches reference site)
- **Icons**: **Lucide React** (modern, tree-shakeable icons)
- **Animations**: **Framer Motion** (smooth, professional animations)
- **State Management**: **Zustand** (lightweight, simple)
- **Form Handling**: **React Hook Form** + **Zod** (validation)
- **HTTP Client**: **Axios**

### Backend
- **Runtime**: **Node.js 20+**
- **Framework**: **Express.js** (lightweight, flexible)
- **CMS/Admin**: **Strapi 4** (Headless CMS - WordPress alternative)
  - Built-in admin panel (like WordPress dashboard)
  - Content types builder (no coding needed)
  - REI API auto-generated
  - Media library
  - User roles & permissions
  - Free and open-source
- **Database**: **PostgreSQL** (robust, relational)
- **ORM**: Built into Strapi
- **Authentication**: **JWT** (JSON Web Tokens)
- **File Upload**: **Multer** + **Cloudinary** (optional)

### DevOps & Deployment
- **Containerization**: **Docker** + **Docker Compose**
  - Frontend container
  - Backend/Strapi container
  - PostgreSQL container
  - Nginx reverse proxy
- **Environment Management**: **dotenv**
- **Process Manager**: **PM2** (optional, for production)

## ğŸ³ Docker Setup

### Services
1. **PostgreSQL Database**
   - Port: 5432
   - Persistent volume for data
   - Environment variables for credentials

2. **Strapi Backend**
   - Port: 1337
   - Connected to PostgreSQL
   - Admin panel: http://localhost:1337/admin
   - API: http://localhost:1337/api

3. **React Frontend**
   - Port: 3000 (development) / 80 (production)
   - Nginx for production serving
   - Environment variables for API URL

4. **Nginx Reverse Proxy** (Optional)
   - Port: 80
   - Routes to frontend and backend
   - SSL/TLS termination (production)

### Docker Compose Configuration
```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    container_name: singleres-db
    environment:
      POSTGRES_DB: singleres
      POSTGRES_USER: strapi
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    networks:
      - singleres-network

  strapi:
    build: ./backend
    container_name: singleres-backend
    environment:
      DATABASE_CLIENT: postgres
      DATABASE_HOST: postgres
      DATABASE_PORT: 5432
      DATABASE_NAME: singleres
      DATABASE_USERNAME: strapi
      DATABASE_PASSWORD: ${DB_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
      ADMIN_JWT_SECRET: ${ADMIN_JWT_SECRET}
      APP_KEYS: ${APP_KEYS}
      API_TOKEN_SALT: ${API_TOKEN_SALT}
    volumes:
      - ./backend:/app
      - strapi_uploads:/app/public/uploads
    ports:
      - "1337:1337"
    depends_on:
      - postgres
    networks:
      - singleres-network

  frontend:
    build: ./frontend
    container_name: singleres-frontend
    environment:
      VITE_API_URL: http://localhost:1337
    ports:
      - "3000:80"
    depends_on:
      - strapi
    networks:
      - singleres-network

volumes:
  postgres_data:
  strapi_uploads:

networks:
  singleres-network:
    driver: bridge
```

## ğŸ“š Technology Justification

### Why Strapi?
- **WordPress-like experience** without WordPress complexity
- **Headless CMS** - API-first, use any frontend
- **Built-in admin panel** - no need to build from scratch
- **Content types builder** - visual interface, no coding
- **Free and open-source**
- **Active community** and good documentation
- **Flexible** - can customize everything
- **Modern** - built with Node.js, supports latest tech

### Why shadcn/ui?
- **Not a component library** - copy components to your project
- **Full control** - own the code, customize freely
- **Built on Radix UI** - accessible by default
- **Tailwind CSS** - consistent with reference site
- **Free** - no licensing issues
- **Modern** - latest React patterns
- **Responsive** - mobile-first design
- **Consistent** - design system out of the box

### Why React + Vite?
- **Fast** - Vite is extremely fast for development
- **Modern** - latest React features (hooks, suspense)
- **Popular** - huge ecosystem, easy to find help
- **Component-based** - reusable, maintainable
- **SEO-friendly** - with proper setup
- **Production-ready** - optimized builds

### Why Docker?
- **Consistency** - same environment everywhere
- **Isolation** - services don't conflict
- **Easy setup** - one command to start everything
- **Portable** - deploy anywhere
- **Scalable** - easy to add more services
- **Professional** - industry standard
